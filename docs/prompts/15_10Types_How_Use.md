## Prompt
### 최종
```
당신은 모바일 앱 시장에서 20년간 활동한 성공적인 성장(Growth) 컨설턴트입니다.

우리의 현재 상황은 다음과 같습니다:
1.  내부 상황: 최근 앱을 출시했으나, 우리 앱의 구체적인 강점(S)과 약점(W) 데이터가 부족한 상황입니다.
2.  외부 상황: 경쟁사가 공격적인 마케팅을 펼치고 있으며, 이로 인해 사용자 증가세가 둔화되고 있습니다.
3.  목표: 다음 분기 마케팅 전략을 새롭게 수립하여 사용자 증가세를 회복시켜야 합니다.

[지시 사항 - 체인 오브 스루트 적용]

현재 정보가 부족하므로, 다음의 3단계 분석 절차를 반드시 따라주세요.

1단계: 강점(S) 및 약점(W)의 전략적 가정 (정보 공백 메우기)
*앱 출시 초기에 우리가 가질 수 있는 가장 논리적인 강점(S) 1가지와 가장 치명적인 약점(W) 1가지를 각각 가정하고 그 이유를 설명하십시오.

2단계: SWOT 전략 방향 선택 및 이유 설명
* 가정한 S, W와 실제 상황 T(경쟁사 공격)를 고려했을 때, 다음 분기에 집중해야 할 가장 현실적이고 효과적인 SWOT 전략 유형(SO, ST, WO, WT 중 1개)을 선택하고, 그 전략을 선택한 컨설턴트로서의 이유를 제시하십시오.

3단계: 핵심 실행 전략 (Tactics) 도출
* 선택한 전략 유형에 따라, 다음 분기에 즉시 실행 가능한 마케팅 전략 3가지를 구체적으로 제시하십시오. 각 전략은 반드시 가정한 S/W를 활용/보완**하고 **T(경쟁사 공격)를 회피 또는 극복하는 데 초점을 맞춰야 합니다.
```
### 1. 지시형(Instructional)
#### prompt

```
#### 미비점
```
정보 입력의 한계: AI가 우리 앱의 구체적인 강점(S)이나 경쟁사의 상세 전략(T)을 모르기 때문에, 답변은 일반적인 '대응 전략' (예: 할인, 바이럴 마케팅)에 머무를 수밖에 없습니다.
```

### 3. 역할 부여(Role-Playing)
#### prompt
```
당신은 20년 경력의 모바일 앱 성장 전략 전문가입니다. 앱 출시 초기에 겪는 경쟁사 공격(T)에 맞서, 다음 분기에 사용자 증가세를 회복시킬 수 있는 성장(Growth) 전략 3가지를 제안해 주세요
```
#### 미비점
```
깊이 없는 실행안: 전문가의 관점과 원칙은 제시되지만, 실제 마케팅 예산(W)이나 앱의 특징(S) 등 구체적인 내부 조건을 모르므로, 실행 단계에서 "어떻게" 할지에 대한 구체적이고 현실적인 아이디어가 부족할 수 있습니다.
```
---
### 4. 제약 조건(Constraint-Based)
#### prompt
```
경쟁사의 마케팅 공격을 1개월 내에 무력화할 수 있는 전략 3가지를 제안하되, 총예산 100만 원 이하로 실행 가능해야 하며, '광고'라는 단어는 사용하지 마세요
```
#### 미비점
```
비현실적인 제약: 공격적인 경쟁 상황(T)에서 '1개월 내 무력화'와 '100만원 이하'라는 극단적인 제약 조건이 충돌하여, 도출된 전략이 실현 가능성이 매우 낮은 아이디어가 될 수 있습니다.
```

```
#### 미비점
```
표면적인 답변: '최근 출시'(S이자 O)의 장점을 일반적인 차원에서만 설명하고, 경쟁사 공격(T)과의 구체적인 연결고리를 찾지 못한 채 답변이 종료될 수 있습니다.
```

### 6. 체인 오브 스루트(Chain of Thought, CoT)
#### prompt
```
경쟁사 공격에 대응하기 위한 다음 분기 마케팅 전략을 도출하세요. 반드시,
'앱의 차별점 분석' 
'타겟 고객 재정의' 
'핵심 메시지 도출'의 3단계 프로세스를 거쳐 최종 전략 2가지를 제시해 주세요.
```
#### 미비점
```
정보 순환 문제: 1단계 '앱의 차별점 분석'에서 실제 정보(S)를 모르므로, AI가 가정을 세우게 되고, 이 가정이 2단계와 3단계까지 이어져 결과 전체의 신뢰성이 저하됩니다.
```

### 7. 페르소나(Persona)
#### prompt
```
당신은 고객의 '고통(Pain Point)'을 해결하는 데 집착하는 괴짜 마케터입니다. 경쟁사 공격을 기회(O)로 역이용하여 사용자 증가세를 급등시킬 파격적인 마케팅 아이디어를 제안해 주세요. 말투는 자유롭고 도발적으로 해 주세요.
```
#### 미비점
```
아이디어의 실용성 저하: '괴짜 마케터'의 역할에 충실한 나머지, 아이디어가 매우 창의적이지만 법적/윤리적 문제가 있거나 현실적인 실행력이 낮은 극단적인 방안이 될 수 있습니다.
```

### 8. 조건부 생성(Conditional)
#### prompt
```
[위협: 경쟁사 공격적 마케팅]과 [조건: 우리 앱은 혁신적인 개인화 기능 보유(S)]를 전제로, 경쟁사의 물량 공세를 뚫고 우리 앱의 개인화(S)를 강조할 수 있는 광고 컨셉 3가지를 생성해 주세요.
```
#### 미비점
```
좁은 시야: '개인화' 기능이라는 하나의 강점(S)에만 갇혀, 만약 그 기능이 실제 사용자에게 크게 어필되지 못한다면 전략 전체가 실패할 수 있는 위험을 간과하게 됩니다.
```
---
### 9. 인스턴스 기반(Instance-Based)
#### prompt
```
'토스(Toss)'가 초기 시장을 장악하기 위해 사용했던 '입소문(바이럴) 전략'을 분석하여, 현재 사용자 증가세가 둔화된 우리 앱에 적용 가능한 벤치마킹 전략 3가지를 제안해 주세요.
```
#### 미비점
```
다른 산업/시장 배경: 토스(금융/핀테크)와 우리 앱의 산업 특성 및 규제 환경(O, T)이 매우 다를 수 있습니다. 따라서 토스 전략의 성공 요인이 우리 앱에는 적합하지 않을 가능성이 높습니다.
```
---
### 10.재구성/번역(Restructuring)
#### prompt
```
우리 앱의 가장 많은 이탈이 발생하는 지점(W)에 대한 솔루션 메시지를, 고객이 '나를 위한 맞춤 앱'이라고 느끼도록 따뜻한 톤으로 재구성해 주세요. (기존 메시지: '데이터 로딩 오류가 자주 발생하니, 잠시 후 다시 시도하세요.')
```
#### 미비점
```
과도한 감성: 기술적인 문제($$W$$)를 해결하는 메시지를 지나치게 감성적으로 재구성할 경우, 고객에게 문제의 심각성을 은폐하는 것처럼 보이거나, 전문성이 떨어지는 느낌을 줄 수 있습니다.
```

[역할 부여: 모바일 앱 성장 컨설턴트의 분석]

#### 1단계: 강점(S) 및 약점(W)의 전략적 가정

| 구분 | 가정된 요소 | 가정 이유 |
| :--- | :--- | :--- |
| 강점 (S) | 최신 기술 기반의 앱 성능 및 사용자 경험(UX) 최적화 | '최근 앱 출시'는 곧 레거시 기술 부채가 없다는 뜻입니다. 경쟁사 앱보다 빠르고 버그가 적으며, 모바일 환경에 최적화된 '기술적 우위'를 초기 강점으로 가정하는 것이 가장 논리적입니다. |
| 약점 (W) | 초기 낮은 브랜드 인지도와 제한된 마케팅 예산 | 신규 앱은 태생적으로 브랜드 파워가 약하며, 사용자 증가세가 둔화된 상황에서 공격적인 경쟁사에 대응할 만한 충분한 마케팅 자금이 부족하다고 가정하는 것이 현실적입니다. |

#### 2단계: SWOT 전략 방향 선택 및 이유 설명

선택된 전략: ST 전략 (강점 - 위협 극복)

현재 가장 시급한 문제는 T (경쟁사 공격)로 인해 W (약점: 낮은 인지도/예산)가 더욱 부각되고 있다는 점입니다. WO 전략(약점 보완 + 기회 활용)은 시간이 오래 걸립니다. 따라서, W를 최소화하고 S (기술적 우위)를 극대화하여 T (경쟁사 공격)를 정면으로 돌파할 수 있는 ST 전략이 가장 현실적입니다.

컨설턴트로서의 이유:

> "지금은 대규모 예산을 투입하는 경쟁사의 물량 공세를 정면으로 따라가기보다, 우리가 가진 유일한 자산인 '최적화된 앱 성능'을 방패(S)로 삼아 위협을 회피해야 합니다. 즉, 광고가 아닌 '경험'으로 승부하여, 경쟁사 광고를 통해 유입된 잠재 고객이 우리 앱을 써보고 '이 앱이 더 좋네'라고 느끼게 만드는 차별화된 포지셔닝이 필요합니다."

#### 3단계: 핵심 실행 전략 (Tactics) 도출

| 전략 유형 | 실행 전략 (Tactics) |
| :--- | :--- |
| ST 전략 | 1. '경험 비교' 기반의 바이럴 캠페인 (S 활용): '우리 앱의 로딩 속도 vs 경쟁사 앱의 로딩 속도' 등 객관적 성능(S)을 보여주는 비교 콘텐츠를 제작하고, 이탈률이 낮은 기존 사용자들에게 리워드를 제공하여 바이럴 확산을 유도합니다. (경쟁사 메시지를 무력화) |
| (S)T 전략 | 2. '최적화된 고객 여정' 기반 온보딩 강화 (W/T 극복): 앱을 처음 설치한 신규 사용자에게 가장 쾌적한 UX(S)를 빠르게 경험하도록 온보딩 과정(W)을 획기적으로 개선합니다. 이탈률을 낮추고(W 보완) 사용자당 획득 비용(CAC)의 효율을 높여 경쟁사 공격(T) 대비 비용 우위를 확보합니다. |
| (S)T 전략 | 3. 핵심 기능 대상 '제로 예산' PR (S 활용): 앱의 가장 뛰어난 기능(S)에 대해 IT, 테크 전문 미디어 및 마이크로 인플루언서들에게 선별적으로 접근하여, 제품력(S)에 기반한 자연스러운 홍보(PR)를 유도합니다. (예산(W)을 아끼면서 경쟁사의 광고성 메시지와 차별화) |
